# Мультитенантная платформа — Спецификация дизайна

**Дата:** 2026-04-01  
**Версия:** v2.0 (Этап 1)  
**Статус:** Согласовано

---

## Контекст

Приложение Detailing Time превращается из одностудийного инструмента в агрегатор студий кар-врапинга. Любая студия в мире может зарегистрироваться, получить хэштег, загружать работы и получать заявки от пользователей через 3D конфигуратор.

---

## Что уже есть (не трогаем)

- Роли `client` / `studio_owner` в таблице `profiles`
- Таблица `studio_profiles` с основными полями
- Таблица `orders` с `studio_id` и `client_id`
- Дашборд студии (заявки, статусы)
- 3D конфигуратор
- Бакет `studio-photos` в Supabase Storage
- Цветовая палитра: золотой `#C9A84C`, тёмный фон `#1a1a1a`

## Структура раздела «Наши работы»

### Уровень 1 — Список студий
- Лёгкий экран без тяжёлых изображений
- Каждая студия — карточка: название, `#hashtag`, город, количество работ
- Фон карточки — CSS-градиент (не фото), цвет акцента уникальный для каждой студии
- Сортировка: сначала студии с наибольшим количеством работ

### Уровень 2 — Галерея студии
- Открывается по тапу на карточку студии
- Сетка **3 колонки** (как Instagram)
- Фото грузятся лениво (lazy loading) по мере скролла
- Supabase CDN отдаёт сжатые превью (thumbnail_url)
- Тап на фото → полноэкранный просмотр
- Кнопка «Отправить заявку в эту студию» внизу экрана

---

## Выбор студии в конфигураторе

При нажатии «Отправить заявку» в 3D редакторе:
- Открывается bottom sheet / экран со списком всех студий
- Каждая студия: название, `#hashtag`, город
- Если пользователь пришёл со страницы студии — эта студия предвыбрана
- Убрать `.limit(1)` в `orders-service.ts`, `studio-service.ts`, `studio/dashboard.tsx`
- Убрать хардкод: email `info@flor-detailing.ru`, телефон, название «Флор»

---

## Пуш-уведомления студии

- При создании заявки → Expo Push API отправляет уведомление студии
- Текст: «Новая заявка — [марка авто], [модель]»
- Токен сохраняется в `push_tokens` при логине студии
- Бесплатно через Expo Push Service

---

## Онбординг новой студии

При регистрации с ролью `studio_owner`:
1. Выбор хэштега (уникальная проверка в реальном времени)
2. Название студии, город, страна
3. Контакты (телефон, Telegram, WhatsApp)
4. Студия сразу активна — верификация не требуется (v2.0)

---

## Хранилище фотографий

- **Storage:** Supabase Storage, бакет `studio-photos`
- **Структура папок:** `studio-photos/{studio_id}/{filename}`
- **CDN:** встроенный в Supabase, глобальный
- **Превью:** при загрузке создаётся thumbnail через Supabase Image Transformation
- **Архивация:** старые фото (>1 года) → Cloudflare R2 ($0.015/GB/мес)
- **Лимит:** начальный план Supabase — 1GB бесплатно

# Галерея студии — Спецификация
**Дата:** 2026-03-31

## Суть
Добавить галерею фотографий работ студии. Карточка «3D Конфигуратор» на главном экране переименовывается в «Наши работы» и ведёт на новый экран галереи. Студийные пользователи могут загружать фотографии с хэштегами через дашборд.

## Scope
Фаза 1 — одна студия (Detailing Time). Агрегатор нескольких студий по хэштегам — в будущем.

## Изменения

### 1. Главный экран (`app/(tabs)/index.tsx`)
- Переименовать карточку: «3D Конфигуратор» → «Наши работы»
- Изменить навигацию: `/catalog` → `/studio/gallery`
- Всё остальное (фото, стиль, размер) без изменений

### 2. Новый экран `app/studio/gallery.tsx` (публичный)
- Доступен всем пользователям без авторизации
- Сетка 2 колонки с фотографиями
- На каждом фото: хэштег студии (`#detailingtime`) + описание при нажатии
- Полноэкранный просмотр по нажатию на фото
- Пустое состояние: «Работы скоро появятся»

### 3. Секция галереи в дашборде (`app/studio/dashboard.tsx`)
- Новая секция «Галерея» для роли studio_owner
- Кнопка загрузки: выбор фото из галереи телефона → загрузка в Supabase Storage `studio-photos`
- Поля: описание (текст) + хэштег (заполняется автоматически `#detailingtime`, только чтение)
- Список загруженных работ с возможностью удаления

### 4. База данных
**Таблица: `studio_works`**
```sql
id          uuid primary key default gen_random_uuid()
studio_id   uuid references studio_profiles(id)
photo_url   text not null
description text
hashtag     text not null default '#detailingtime'
created_at  timestamptz default now()
```

**Storage:** Supabase bucket `studio-photos` — публичное чтение, запись только авторизованным

### 5. Сервис (`constants/studio-works-service.ts`)
- `getStudioWorks()` — получить все работы, сортировка по дате (новые сверху)
- `uploadStudioWork(photoUri, description, hashtag)` — загрузить фото + добавить запись
- `deleteStudioWork(id)` — удалить запись + файл из storage

## Поток данных
1. Владелец студии открывает дашборд → нажимает «Добавить фото»
2. Выбирает фото из галереи → загружается в bucket `studio-photos` → запись в `studio_works`
3. Клиент на главном экране нажимает «Наши работы» → `/studio/gallery`
4. Галерея загружает `studio_works` из Supabase → отображает сетку фотографий

## Не входит в эту версию
- Фильтрация по хэштегам нескольких студий (будущее)
- Загрузка видео (будущее)
- Комментарии и лайки (будущее)
